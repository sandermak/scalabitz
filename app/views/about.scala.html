About Scalabitz

Scalabitz shows Scala-related content that is currently popular on the web. It polls the Bit.ly API to find trending links. Once published on Scalabitz, the click-count for a link is updated periodically through the Bit.ly API. Links are shown in the order they are published.

Publishing links is semi-automated: Scalabitz retrieves candidate links which are automatically scored on relevance. However, publishing to the frontpage is a manual action. The relevance scoring is done crudely and needs some improvement before I trust it to automatically push links to the frontpage...

Watch my blog to see how Scalabitz progresses.
Under the hood

Of course Scalabitz is built using Scala, with Play 2.1. MongoDB is used to store the data. The Play webapp accesses Mongo with the ReactiveMongo driver. This driver is fully asynchronous and meshes well with Play's async architecture. Calls to the Bit.ly API are also fully asynchronous using Play's WS API. They are trigger by scheduled messages to Akka actors.

Scalabitz is currently deployed on Heroku and Mongo is hosted by MongoHQ.
