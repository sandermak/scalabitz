About Scalabitz

Scalabitz shows Scala-related content that is currently shared on the web. It polls the Bit.ly API to find relevant links. Once published on Scalabitz, the click-count for a link is updated periodically through the Bit.ly API. Links are shown in the order they are published. Read this blogpost for a more detailed introduction to the project.

In the near future Scalabitz will also publish new links directly to Twitter. Don't miss and start following:

Follow @@scalabitz
Curation

Publishing links is semi-automated: Scalabitz retrieves candidate links which are automatically scored on relevance. However, publishing to the frontpage is a manual action. The relevance scoring is done crudely and needs some improvement before I trust it to automatically push links to the frontpage...

Watch my blog to see how Scalabitz progresses.
Under the hood

Of course Scalabitz is built using Scala, with Play 2.1. MongoDB is used as datastore. The Play webapp accesses Mongo using the ReactiveMongo driver. This driver is fully asynchronous and meshes well with Play's async architecture. Calls to the Bit.ly API are also fully asynchronous using Play's WS API. Akka actors are used for all the scheduled jobs.

Scalabitz is currently deployed on Heroku and Mongo is hosted by MongoHQ.
